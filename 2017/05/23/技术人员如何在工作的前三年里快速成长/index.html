仁梓小舍

技术人员如何在工作的前三年里快速成长

2017-05-23

作者/分享人：Phodal

摘要

  • 基础篇：正确练习姿势。从练习使用快捷键到重构，再到使用新的框架练习。

  • 进阶篇：通过练习提高。初学时，使用Vue，React去高仿一些项目；有经验以后，高仿应用只会让我们更累。我们需要一些高级的练习技巧，从引入别的框架思想，到造各种各样的轮子。

基础篇

打字练习

打字练习软件：Typing

目的：为了练习准确度和打字的速度。

练习一个月，每天大概半小时。

练习项目：

  • 打字姿势与技巧
  • 精通IDE 与编辑器的快捷键
  • 命名技巧和重构代码技巧掌握好

语言框架的练习

  1. 阅读：买相关书籍、查阅教程和官方指南
  2. 模仿：找个合适的Demo，熟悉基础概念
  3. 扩展：在Demo基础上，实现业务功能，了解功能、特性
  4. 补漏：查看官方文档，看有没有漏掉什么东西。
  5. 记录：撰写博客、日志记录过程

进阶篇

工程实践：模仿开源软件

工作时通常写业务代码，纯技术实践并不多。工作经验与技术能力不关联。

在学校写得代码与工作写得代码，区别在于：软件工程实践。
软件工程实践包括：

  • 使用版本管理。诸如Github上的Git。
  • 使用持续集成。为团队写作，提供可靠的帮助。
  • 完整的测试用例。边写单元测试，功能测试。
  • 代码检视。用于提高整个代码的质量。

以GITHUB 优秀开源软件React为例，其拥有下面几个badge:

  1. Circle CI, 持续继承，诸如测试是否都通过，部署是否成功。
  2. Travis CI 同上。
  3. Coverage，代码测试覆盖率
  4. npm 当前版本号
  5. PRs welcome，欢迎来到Pull Request。

那么我们在实践的时候，就可以模拟这样的项目组成，一步步往下实践：

  1. 为项目添加测试框架，如Java里的 Junit， Node.js里的 Mocha 等等。
  2. 添加自动化测试脚本，如Java里的Gradle， Node.js里的Grunt、Gulp、NPM等等。
  3. 添加测试覆盖率工具
  4. 添加持续集成，如Travi CI 或者Circle CI。
  5. 添加代码质量管理工具，如 Code Climate。
  6. 制定目标，并完成。
